## Summary of Animation Control

### What controls what:

| Component | Responsibility | Triggered By |
|-----------|---------------|--------------|
| **TransitionContext** | Exit animations (overlay + page scale) | `startTransition(url)` called |
| **TransitionWrapper** | Locomotive Scroll, listens for nav events | Mounts once in root layout |
| **PageTransition** | Page enter animations (slide down + fade) | New page component mounts |
| **Header/Nav** | Menu container slide-in | `isActive` state changes to true |
| **Header/Link** | Individual link animations | Nav component mounts |
| **Header/Curve** | SVG path morph | Nav component mounts |

### Animation Coordination:

1. **Exit Phase** (TransitionContext): Overlay slides up, page scales down
2. **Navigation** (TransitionWrapper): Listens for custom event, triggers router
3. **Enter Phase** (PageTransition): New page slides down, content fades in
4. **Reveal Phase** (TransitionContext): Overlay slides down to reveal page

All animations use GSAP for consistent timing and smooth 60fps performance.
